# Customer Churn Prediction using Artificial Neural Network (ANN)

A deep learning project that predicts customer churn for a telecommunications company using a fully connected Artificial Neural Network (ANN) built with TensorFlow/Keras.

---

## Dataset

**Source:** [Telco Customer Churn — Kaggle](https://www.kaggle.com/datasets/blastchar/telco-customer-churn)

The dataset contains **7,043 customer records** with **21 features** covering demographics, subscribed services, account information, and a binary churn label (`Yes`/`No`).

| Feature Category | Examples |
|---|---|
| Demographics | `gender`, `SeniorCitizen`, `Partner`, `Dependents` |
| Services | `PhoneService`, `InternetService`, `StreamingTV`, `OnlineSecurity` |
| Account Info | `Contract`, `PaymentMethod`, `MonthlyCharges`, `TotalCharges`, `tenure` |
| Target | `Churn` (Yes / No) |

---

## Project Workflow

### 1. Environment Setup
- Installs `keras-tuner` for hyperparameter optimization.
- Imports standard libraries: `numpy`, `pandas`, `matplotlib`, `seaborn`, `scikit-learn`, `tensorflow`, `keras`, and `keras_tuner`.
- Seeds are fixed (`SEED = 42`) to ensure reproducibility.
- Dataset is loaded from Google Drive via Google Colab.

### 2. Exploratory Data Analysis (EDA)
- **Shape & Info:** 7,043 rows × 21 columns. Mix of object, int64, and float64 types.
- **Missing Values:** No null values detected.
- **Duplicates:** No duplicate records found.
- **Numerical Stats:** Range, mean, median, and standard deviation computed for `tenure`, `MonthlyCharges`, and `SeniorCitizen`.
- **Visualizations:**
  - Churn distribution (class imbalance identified — ~73% No, ~27% Yes).
  - Scatter plot of Tenure vs. Monthly Charges segmented by churn status to reveal behavioral patterns.
  - Additional EDA plots to understand feature distributions and relationships.

### 3. Data Preprocessing
- **Target Encoding:** `Churn` column mapped from `Yes/No` → `1/0`.
- **Feature Encoding:** Categorical columns one-hot encoded using `pd.get_dummies(drop_first=True)`, expanding features from 21 to **36 columns**.
- **Class Imbalance Handling:** **SMOTE** (Synthetic Minority Oversampling Technique) applied via an `imblearn` pipeline with median imputation, balancing the churn classes before training.
- **Train/Test Split:** Standard 80/20 split.
- **Feature Scaling:** `StandardScaler` applied to normalize numerical features.

### 4. Model Architecture
A Sequential ANN built with Keras:

```
Input Layer  →  Dense(32, ReLU)  →  Dropout(0.2)
             →  Dense(16, ReLU)  →  Dropout(0.2)
             →  Dense(1, Sigmoid)
```

- **Optimizer:** Adam
- **Loss Function:** Binary Crossentropy
- **Regularization:** Dropout layers (rate = 0.2) to prevent overfitting
- **Early Stopping:** Monitors `val_loss` with `patience=5` and restores best weights
- **Training Config:** Up to 100 epochs, batch size of 32, 20% validation split

### 5. Model Training & Convergence
Training converged around **epoch 15–19**, with both training and validation loss declining smoothly. Key observations:
- No sign of overfitting — validation loss plateaued without rising.
- Healthy convergence with final loss in the range of ~0.40–0.45.
- Smooth, balanced loss curves — indicative of a well-behaved ANN.

### 6. Evaluation & Results

| Metric | Score |
|---|---|
| **Accuracy** | **81.88%** |
| **Precision** | 0.7868 |
| **Recall** | 0.8756 |
| **F1 Score** | 0.8288 |
| **ROC AUC** | 0.8187 |

**Confusion Matrix:**
```
              Predicted No   Predicted Yes
Actual No        787             246
Actual Yes       129             908
```

The high **Recall (87.6%)** means the model successfully catches most customers who actually churn — critical for business retention strategies.

An **ROC curve** is also plotted to visualize the trade-off between true positive and false positive rates across thresholds.

##  Where to Use This Model

This churn prediction model is well-suited for:

- **Telecom Companies** — Identify at-risk subscribers before they cancel, enabling targeted retention campaigns.
- **SaaS & Subscription Businesses** — Flag customers likely to downgrade or leave and trigger proactive outreach.
- **Customer Success Teams** — Prioritize high-risk accounts for dedicated support or discount offers.
- **Marketing Automation** — Feed churn scores into CRM tools (Salesforce, HubSpot) to trigger personalized win-back flows.
- **Product Analytics** — Identify which service features (e.g., lack of `TechSupport`, `Month-to-month` contracts) are most predictive of churn.
- **Finance & Risk** — Forecast revenue loss due to expected churn and plan accordingly.

##  Future Enhancements

| Enhancement | Description |
|---|---|
| **Hyperparameter Tuning** | Leverage the already-installed `keras-tuner` to run Bayesian or Hyperband search over layer sizes, dropout rates, learning rates, and batch sizes. |
